# Domain Specialist Selection Guide
Revision: 1.0
## Usage
1. Identify project's primary focus
2. Match to domain specialist
3. Load Core-Developer-Instructions.md + selected Domain

## Decision Tree
| Focus | Specialist |
|-------|------------|
| User Interfaces | Frontend-Specialist |
| APIs/Server Logic | Backend-Specialist or API-Integration-Specialist |
| Deploy/Operate | DevOps-Engineer or SRE-Specialist or Platform-Engineer |
| Data Pipelines | Data-Engineer |
| Database Optimization | Database-Engineer |
| Mobile Apps | Mobile-Specialist |
| Security | Security-Engineer |
| System Design | Cloud-Solutions-Architect |
| Quality Assurance | QA-Test-Engineer |
| Hardware/Firmware | Embedded-Systems-Engineer |
| Machine Learning | ML-Engineer |
| Performance | Performance-Engineer |
| Requirements/PRD | PRD-Analyst |
| WCAG Compliance | Accessibility-Specialist |

## Quick Reference
| Domain | Primary Focus | Example Use Cases |
|--------|---------------|-------------------|
| Backend | Server-side, APIs | REST APIs, microservices |
| Frontend | UI, client-side | SPAs, web apps |
| DevOps | Deployment, infra | CI/CD, containers |
| Database | Data modeling | Schema, query tuning |
| API-Integration | Integration | API gateways, event-driven |
| Security | App security | Audits, compliance |
| Platform | Dev platforms | Internal portals |
| Mobile | iOS/Android | Native/cross-platform |
| Data | Pipelines | ETL, analytics |
| QA & Test | Testing | Test automation |
| Cloud Architect | Architecture | Cloud, scalability |
| SRE | Reliability | SLOs, incidents |
| Embedded | Firmware | IoT, microcontrollers |
| ML | Machine learning | Models, MLOps |
| Performance | Optimization | Load testing |
| PRD Analyst | Requirements | PRDs, stories |
| Accessibility | WCAG, AT | Audits, compliance |

## Example Combinations
**Full-Stack Web:** Core + Backend + Frontend + Database
**Cloud-Native Microservices:** Core + API-Integration + DevOps + Cloud-Architect
**Mobile App + Backend:** Core + Mobile + Backend
**Data Platform:** Core + Data-Engineer + Database + Cloud-Architect
**Secure Production:** Core + Backend + Security + SRE
