## Platform Coverage
- Godot Engine: 2D/3D with GDScript or C#
- Unity: 2D/3D with C#
- Unreal Engine: 3D with C++ or Blueprints
- Browser Games: Phaser, PixiJS, Three.js

## Game Types
2D (platformers, puzzles, roguelikes), 3D (FPS, racing), Mobile, Browser, Multiplayer

## Initialization Questions
- Game engine?
- 2D or 3D?
- Game genre?
- Target platform?
- Art style?

## Engine Selection
| Engine | Best For |
|--------|----------|
| Godot | Open source, 2D/3D, beginners |
| Unity | Mobile, 3D, large ecosystem |
| Unreal | High-fidelity 3D, AAA |
| Phaser | Browser 2D games |

## Vibe Patterns
1. **Core Loop First:** Get one thing playable before polish
2. **Placeholder Art:** Use shapes/colors initially
3. **Frequent Play-testing:** Run and play after every change
4. **Iteration Over Perfection:** Try ideas quickly

## Verification Pattern
```
STEP 6: Run game in engine editor
STEP 7: Play-test the mechanic
STEP 8: Note what feels right/wrong
STEP 9: Report gameplay experience
```

## Game-Specific Triggers
- Multiple interacting systems → Evolution
- Need save/load → Evolution
- Need multiplayer → Evolution
