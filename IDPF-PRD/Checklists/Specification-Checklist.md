# Specification Phase Checklist

## Requirement Detailing
For each requirement:
- [ ] REQ-ID assigned
- [ ] Description clear and unambiguous
- [ ] Rationale documented
- [ ] Priority assigned
- [ ] Dependencies identified

## Acceptance Criteria
For each requirement:
- [ ] Given/When/Then defined
- [ ] Happy path covered
- [ ] Error scenarios covered
- [ ] Edge cases identified
- [ ] Criteria are testable

## NFR Specifications
For each NFR:
- [ ] Measurable target defined
- [ ] Measurement method specified
- [ ] Testing approach defined
- [ ] Linked to functional requirements

## Testing Approach
- [ ] TDD confirmed (required)
- [ ] ATDD decision (yes/no + rationale)
- [ ] BDD decision (yes/no + rationale)
- [ ] Test frameworks selected
- [ ] Coverage targets defined

## Traceability
- [ ] Requirements linked to features/NFRs/constraints
- [ ] Traceability matrix (if Comprehensive)

## Phase Gate
- [ ] All requirements detailed
- [ ] Technical/business review complete
- [ ] Ready for Generation
