# Construction/

Tactical artifacts created during active development. This directory is populated as work progresses—it starts empty and grows organically.

## Subdirectories

### Test-Plans/

**Purpose:** Proposed test cases and test strategies for specific features or components.

**When populated:** When testing approach is discussed or planned during development.

**Template:** `Test-Plan-{feature}.md`

---

### Design-Decisions/

**Purpose:** Technical decisions made during the build, with context and rationale.

**When populated:** When architectural or design choices are made that should be documented.

**Template:** `Decision-{NNNN}-{title}.md` (numbered for chronology)

---

### Sprint-Retros/

**Purpose:** Retrospective summaries from completed sprints/iterations.

**When populated:** After `/microsprint close` or end of sprint.

**Template:** `Retro-{sprint-name}.md`

---

### Tech-Debt/

**Purpose:** Identified technical debt for later resolution.

**When populated:** When work is deferred or shortcuts are noted.

**Template:** `Debt-{title}.md`

---

## Usage Notes

- Files are created by the AI assistant when triggers are detected
- All files should be committed to version control
- This directory reflects the "how" of building—tactical decisions during construction
- See `Inception/` for strategic "what" decisions made upfront
