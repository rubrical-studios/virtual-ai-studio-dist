# Deployment Awareness
**Version:** v0.2.0

---

## Overview
`process-docs` (dev) → `virtual-ai-studio-dist` (distribution for users)

## Deployment Architecture

```
Source Files → /minimize-files → .min-mirror/ → deploy-dist.yml → -dist repo
                                                (on git tag v*)
```

**Trigger:** Git tag `v*` pushes to `-dist` via GitHub Actions.

## File Classification

### Deployed FROM `.min-mirror/`
| Directory | Content |
|-----------|---------|
| `IDPF-*` | All 12 framework directories |
| `Overview/` | Framework summaries |
| `Reference/` | GitHub Workflow, Session Startup |
| `Skills/` | Skill definitions and resources |
| `System-Instructions/` | Core and Domain specialists |
| `Assistant/` | Anti-hallucination rules (subset) |

### Deployed AS-IS (Not Minimized)
`README-DIST.md`, `CHANGELOG.md`, `LICENSE`, `install.js`, `fetch-updates.js`, `framework-manifest.json`, `Templates/`

### Dev-Only (Never Deployed)
- **Directories:** `Archive/`, `Proposal/`, `PRD/`, `Guides/`, `Market-Analysis/`
- **Files:** `Skills/MAINTENANCE.md`, maintainer commands
- **Patterns:** Files with "Merge", `Skills/Packaged/`
- **Build:** `.claude/rules/` (generated by install.js for users)

## Issue Workflow Integration

**Before implementing, ask:**
1. Does this affect user-facing files? → Run `/minimize-files`
2. Is this dev-only? → No minimization needed
3. Does this add new directories? → Update `deploy-dist.yml`

**Pre-Commit for User-Facing Changes:**
- [ ] Source files updated
- [ ] `/minimize-files` run
- [ ] `.min-mirror/` changes reviewed
- [ ] `deploy-dist.yml` updated (if new directories)

## Key Commands

| Command | Purpose |
|---------|---------|
| `/minimize-files` | Update `.min-mirror/` |
| `/minimize-files orphans` | Find orphaned files |
| `/minimize-files reset` | Force full rebuild |
